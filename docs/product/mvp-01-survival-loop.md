# MVP 01: Survival Loop

## 最小可用闭环目标

这是我们的“第一块积木”，只做最底层的存活验证。

*   **世界生成**：地图上有 3–10 个小人
*   **核心核心**：只有 hunger (饥饿) 与 thirst (口渴)
*   **可用资源**：只有 berry (浆果) 与 water (水资源单元)
*   **行为边界**：小人仅具备 感知、移动、采集、吃、喝 这五项能力
*   **视图投射**：UI 只要能看见实时状态和跑马灯事件就行
*   **【绝对禁区】**：暂时绝对不做社交、不做信仰、不做科技、不接任何 LLM

没有基于这种确定性的跑通，后面每个 patch 都会往“大而全”漂。
