# Golden Scenario 001

这是我们的“唯一真理”零号回归场地。所有的第一阶段系统跑通必须基于此场景绝对确定复现。

## 场景配置
- **Seed**: `123456789`
- **Map Size**: `20x20` 平面网格，左上角 `(0,0)`
- **Agents**: 开局恰好生成 `5` 个 `species: 'human'`
- **Nodes**: 
  - 恰好生成 `2` 个 berry 资源点
  - 恰好生成 `1` 个 water 水源点
- **Terrain Obstacle**: 存在 `1` 块高 moveCost 的泥沼地形。
- **Initial State**: 
  - 1 个 agent 故意处于极度口渴状态 (thirst: 90)
  - 1 个 agent 处于中度饥饿状态

## 回归校验目标
1. 小人寻找资源的寻路逻辑会不会死循环、卡墙。
2. 那个口渴极高的小人会不会第一时间走向水点，还是错误地去了浆果点。
3. 运行 **300 tick** 之后，这 5 个人的最终存活比例是多少？是否有事件日志崩坏？

> 本文档非开发指南，而是**集成验证标的**。未来 CI 系统应能自动读取此配置运行 `pnpm test:golden`。
