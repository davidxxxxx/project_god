# Balance Tuning Workflow

Use this workflow when changing rates, probabilities, thresholds, or rewards.

## Required approach
1. State the current problem in gameplay terms.
2. State the hypothesis for the tuning.
3. List exact parameters changed.
4. Predict expected behavioral change.
5. Note possible side effects.

## Metrics to inspect
- survival time
- food security stability
- idle rate
- action diversity
- invention rate
- ritual frequency
- social conflict frequency
- runaway snowball risk

## Final output
- parameters changed
- intended impact
- observed impact
- what still needs tuning
